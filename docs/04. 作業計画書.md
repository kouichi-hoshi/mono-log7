# フェーズ0
- cursor設定
	- [AGENTS.md](http://AGENTS.md)を設定する
# フェーズ1
- 開発環境構築
	- フレームワーク、ライブラリをインストールする
		- 02. 仕様書のアーキテクチャを参照
 - 基本のUIパーツ作成
	 - button / dialog / alert / spinner など、未ログイン画面・ログイン中画面で共通して使う shadcn/ui コンポーネントを用意する
 - レイアウト構築（フェーズ1完了条件）
	 - 未ログイン画面
		 - ヘッダー（アプリ名＋ログインボタン）
		 - メイン（ウェルカムメッセージ＋ログインボタン）
	 - ログイン中画面（スタブまたは簡易実装でよい）
		 - ヘッダー（ユーザーアイコン＋ログアウトリンク）
		 - メイン（エディタ＋投稿一覧タイムラインの静的レイアウト）
		 - フッター（表示切り替えコンポーネントと「ゴミ箱を見る」リンク）
 - 最低限の振る舞い
	 - ログインボタン押下でログインモーダルが開く（実際の認証処理は後続フェーズで拡張）
	 - エディタで入力したテキストをローカル／スタブに保存・一覧に反映できる
	 - スピナーの表示・非表示が視覚的に確認できる
	 - 基本的なトースト（成功／エラー）が表示できる

# フェーズ2
- DB接続
- ユーザー認証
- タイムライン機能の拡張
	- 投稿一覧のソート機能（投稿日／更新日、昇順／降順）
	- 無限スクロールを本番の DB + Server Actions + TanStack Query で実現する
- アカウント管理
	- アカウント削除機能（DB のユーザーデータ削除、セッション終了、完了メッセージ表示）
- タグ・外観
	- 投稿保存時のタグ付与（初期値 null、基本的な追加／削除／ラベル編集）
	- 外観変更（システム／ライト／ダーク、文字サイズの変更）
- テストの充実
	- 上記機能に対する単体テスト・結合テストを追加する
	- 詳細なテスト内容は 03. 設計書の「テスト戦略」を参照

# フェーズ3
- 検索・整理機能の強化
	- テキスト検索（タイトル／本文／カテゴリ／タグを対象）
- タグ機能の拡張
	- カスタムタグ追加／編集／削除
	- タグ削除時の挙動（タグが付いていた投稿は「タグ未登録」とする）
- 投稿のライフサイクル管理
	- ゴミ箱投入日から一定日数経過後の自動削除
- 投稿エクスポート（検討）
	- 具体的なエクスポート形式・操作は今後検討（TODO）

※ 各フェーズの詳細な画面仕様は `02. 仕様書.md`、内部構造・テスト方針は `03. 設計書.md` を参照すること。
