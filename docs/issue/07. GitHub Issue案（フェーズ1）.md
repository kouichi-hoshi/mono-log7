## 目的

- フェーズ1（スタブモードでUIの一通りの操作が確認できる状態）を、GitHub Issueとして管理しやすい粒度に分割する。
- 原則: **1 Issue = 1〜2日で完了 / PR1本でクローズ可能**。
- 参照元: `docs/05. チェックリスト式実装計画書.md` の **☐（未着手）**。

## 共通運用（おすすめ）

- **Milestone**: `Phase 1`
- **Labels（例）**:
  - `phase-1`
  - 領域: `ui` / `layout` / `auth` / `adapter` / `editor` / `list` / `filter` / `trash` / `error`
  - 品質: `test`

## Issue 分割案（フェーズ1）
